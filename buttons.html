按钮设计 · 三方案预览

鼠标悬停 / 点击每个按钮可测试交互状态。选好后告诉我哪套，我来批量替换主应用。

方案 A
微立体
主按钮有 amber 光晕 shadow + 顶部高光 inset，hover 上浮 1px，click 下压 1px。次按钮细边框 + 极淡 inset。 整体专业感强，适合金融工具，不抢 focus、不花哨。

工具栏 context
适合场景：整个 app 的主基调。主按钮有分量感但克制，次按钮不抢戏。amber glow 与品牌色高度一致，dark/light 两套都好看。
方案 B
圆润 SaaS
border-radius 10px，padding 更宽松。hover 有轻微 scale(1.03) 弹出感，click 有 scale(0.97) 回弹。 风格偏 Linear / Vercel，更现代。按钮更"大"，手机上点击舒适度更高。

工具栏 context
适合场景：更活泼、更 consumer-friendly。如果目标用户是年轻散户这套很对。信息密度高的区域（持仓卡片里的小操作按钮）会稍显拥挤。
方案 C
分级清晰
三级明确层次：主按钮填充色 + shadow，次按钮 amber outlined（tint 底色 + accent 边框），ghost 完全无边框纯文字。 hover 改颜色深浅而非形变。视觉层次最清晰，适合 UI 复杂区域。
