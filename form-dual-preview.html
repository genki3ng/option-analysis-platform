子批 C · UX 大改：表单双轨模式

推荐表单：目标驱动 vs 策略驱动

现在的表单是策略驱动（你选 direction × intent × timeframe × risk → 看候选）。 需求是加一条目标驱动路径（你说想拿多少租金 / 想接货 / 最大化安全度 → 系统倒推参数）。 下面 3 个 UX 方向，每个都展示桌面 + 手机两个视口。挑一个我落地。

📋 现状 · baseline

当前生产版只有策略驱动这一轨。direction × intent × timeframe × risk 四段 + 5 个预设按钮。

C · 一句话提问 + 快捷 chips — 自然语言入口，最贴近聊天

打开 modal 顶部一个 自然语言输入框「我想…」+ 下面 chips 是最常见目标的一键填充。 输入后用规则 parser 解析 ticker / $ / 安全度 等 → 倒推参数 → 显示 parsed 结果让用户确认。 实在解析不了 → fallback 链接「→ 切换到策略驱动表单」。 对老手快（敲一句就跳）；对新手友好（chips 是 fallback）； parser 失败的容错很重要。

✓ 开发量适中

✓ Mode 分得最清

× 难加默认偏好（"我老用目标驱动"）

✓ 老手最快
✓ 像聊天工具
× Parser 复杂度高
× 解析失败时体验差
