方案 A

新闻播报式 · Headline + 行动按钮

头版 3 条 + 底部 chips。每天提炼最值得看的 3 件事（财报来袭 / 锁利机会 / 行权风险）， 每条带一个一键行动按钮（→ 看持仓 / → 推荐对冲）。底部 chips 是当日健康指标。
优势：极简、扫读体感最强、最像"晨刊摘要"。 取舍：每天只能展示 3 条，规则排序需要算法决定取哪 3 条，可能漏；指标 chips 信息密度低。

方案 B

Dashboard 式 · KPI + 新事件 + 今日该做 + 日历

4 格 KPI + 双列分区 + 14 天日历条。左列是"📢 新事件 since 昨天"（财报披露日变化 / 新买入 / 跨越 80% 利润等"质变"事件）， 右列是"🎯 今日该做"（≤3 条 actionable）。底部时间线一眼看到未来 14 天哪天有财报哪天到期。
优势：信息量最大、给"活体新信号"留足空间、未来 14 天可视化清晰。 取舍：信息密度高，"30 秒扫读"目标压力大，手机上要堆叠分区。

方案 C

对话式 · 包租公管家 (LLM 摘要 + 一键展开)

头像 + 自然语言一段话 + 快速 chip + 展开数据。后端把所有信号喂给一个 LLM (Claude Haiku，便宜)， 生成 2-3 句"今早重点"摘要，附 3 个 chip 让用户跳到对应详情；想看数字点"展开数据"。
优势：最人性化、最简洁、能根据用户历史（trade journal、过去操作）做真正个性化建议。 取舍：LLM 调用要 API key + 每日成本（每用户约 $0.001/天 = $0.36/年，可忽略但要预算）；非确定性，偶尔 prompt 会输出奇怪内容；冷启动慢 1-2 秒。
